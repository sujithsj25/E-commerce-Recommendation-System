3) METHODOLOGY
================================================================================
The project followed a clear step-by-step process to ensure everything works together perfectly:

The development process was executed through a rigorous, multi-phased technical lifecycle to ensure high accuracy and system performance:

- **Phase 1: Behavioral Telemetry Acquisition**: The system implements a robust interaction tracking layer that captures granular user actions, such as "Likes" and cart additions. These events are persisted as relational data points, forming the multi-dimensional dataset required for machine learning analysis.
- **Phase B: Vector Space Modeling & Similarity Analysis**: Using the Pandas and Scikit-Learn libraries, the system transforms raw database entries into a high-dimensional User-Product matrix. It then applies the **Cosine Similarity algorithm** to measure the angular distance between user vectors, identifying "taste neighbors" with mathematical precision.
- **Phase C: Predictive Ranking & Collaborative Filtering**: Once user similarities are established, the system executes a collaborative filtering pass to identify products highly rated by similar users but unseen by the target user. These products are then ranked based on a calculated "Preference Score."
- **Phase D: Asynchronous State Management via HTMX**: To eliminate page-load latency, the recommendation logic is served through an asynchronous endpoint. The system uses **HTMX** to dynamically inject these results into the DOM, ensuring a high-performance, Single Page Application (SPA) experience.
- **Phase E: Performance Optimization & Validation**: The backend logic was optimized using Django's `select_related` and `prefetch_related` methods to minimize database hits. The ML pipeline was further tuned for millisecond response times, ensuring the recommendations remain relevant and fast even as the catalog scales.

This structured engineering methodology ensures that the platform is computationally accurate, secure, and provides a seamless user journey.

import pandas as pd
from sklearn.metrics.pairwise import cosine_similarity
from .models import UserInteraction

def get_recommendations(user_id):
    """
    Main AI engine that calculates User-Based Collaborative Filtering.
    Uses Cosine Similarity to find taste neighbors and predict product preference.
    """
    # 1. Data extraction from the relational schema
    data = list(UserInteraction.objects.all().values())
    if not data: return []

    # 2. Construction of the User-Item Pivot Table (High-Dimensional Matrix)
    df = pd.DataFrame(data)
    matrix = df.pivot_table(index='user_id', columns='product_id', values='rating').fillna(0)

    # 3. Early exit if the user is new (Cold Start)
    if user_id not in matrix.index: return []

    # 4. Execution of the Cosine Similarity Algorithm (Vector Space Analysis)
    similarity = cosine_similarity(matrix)
    sim_df = pd.DataFrame(similarity, index=matrix.index, columns=matrix.index)

    # 5. Identification of nearest neighbors and weighted preference ranking
    neighbors = sim_df[user_id].drop(index=user_id)
    neighbors = neighbors[neighbors > 0].sort_values(ascending=False)

### C. DATABASE STRUCTURE (store/models.py)
This defines how products and user actions are stored.

```python
from django.contrib.auth.models import User
from django.db import models

class Product(models.Model):
    name = models.CharField(max_length=200)
    category = models.CharField(max_length=100)
    price = models.FloatField()
    image = models.ImageField(upload_to='products/')

class UserInteraction(models.Model):
    user = models.ForeignKey(User, on_delete=models.CASCADE)
    product = models.ForeignKey(Product, on_delete=models.CASCADE)
    rating = models.FloatField(default=1) # Records the user's preference
```
